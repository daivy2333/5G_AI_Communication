| 类别 | 准确率 |
|------|--------|
| BPSK | 99.2% |
| QPSK | 91.4% |
| 16QAM | 79.8% |

问题集中在 64QAM/256QAM，而不是训练流程整体不可用。

## 3. 当前实现观察

高阶 QAM 的生成逻辑是正确归一化的矩形星座，但分类输入只是一段经过脉冲成形、多径、相位噪声、DC offset、AWGN 后的 IQ 序列。模型结构是通用三层 CNN，没有显式使用星座统计特征，也没有对 64QAM/256QAM 做难例加权。

当前训练设置是：

| 参数 | 当前值 | 来源 |
|------|--------|------|
| 每类训练样本 | 1000 | `collect_experiment_data.py:224` |
| epoch | 50 | `collect_experiment_data.py:225` |
| batch size | 128 | `collect_experiment_data.py:225` |
| learning rate | 1e-3 | `collect_experiment_data.py:225` |
| dropout | 0.4 | `collect_experiment_data.py:226` |
| 测试样本 | 每类 500 | `collect_experiment_data.py:231` |

训练日志显示 50 epoch 后训练准确率约 75.6%，验证最佳 69.0%。这更像容量/数据分布/高阶类可分性问题，而不是过拟合到训练集。

## 4. 优先改进方向

### 4.1 增加高阶 QAM 样本与分层评估

最小可验证方案是把 T07 拆成两个实验：

1. 保持 5 类均衡，但把 `n_per_class` 从 1000 提高到 3000 或 5000。
2. 增加 64QAM/256QAM 的过采样实验，同时保留独立均衡测试集。

验收指标应分别看 `class_64QAM` 和 `class_256QAM`，不要只看 overall accuracy。

### 4.2 增加 SNR 分层结果落盘

`CNNModulationRecognizer.evaluate()` 已支持 `accuracy_by_snr`，但 `collect_modulation_recognition()` 目前没有把 SNR 分组写入 CSV。T07 需要判断高阶 QAM 是全 SNR 都差，还是低 SNR 拖累，因此建议新增 `modulation_recognition_by_snr.csv`。

### 4.3 引入星座/幅度统计辅助特征

已有 `_extract_features()` 的统计特征备用逻辑包含幅度均值、方差、分位数、相位统计、高阶矩。CNN 路径没有使用这些特征。后续可做双分支：IQ CNN 分支 + 统计特征 MLP 分支，尤其服务于 64QAM/256QAM 的幅度层级区分。

### 4.4 避免重复踩坑

已有 L04/L05 表明数据泄漏和未 shuffle 曾经严重影响调制识别。T07 的任何提升实验都必须保留：

| 约束 | 原因 |
|------|------|
| 训练/验证切分前打乱 | 防止验证集只包含最后一个类别 |
| 训练与测试不复用固定 seed=idx | 防止测试样本与训练样本重复 |
| 独立测试集每类样本数一致 | 避免 overall accuracy 掩盖高阶 QAM 失败 |
